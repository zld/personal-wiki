## 能力对照

| 维度                  | Smalltalk            | Objective-C  | 是否保留  | 说明       |
| ------------------- | -------------------- | ------------ | ----- | -------- |
| 消息发送                | 一切皆消息                | `[obj sel]`  | ✅     | 语义保留     |
| 方法缺失处理              | `doesNotUnderstand:` | 三段转发         | ⚠️    | 语义拆解     |
| 运行时改类               | 任意修改                 | 部分支持         | ⚠️    | 受 ABI 限制 |
| Method Dictionary   | 动态可变                 | cache + list | ⚠️    | 性能优先     |
| 元类系统                | 统一对象模型               | 元类保留         | ✅     | 核心未丢     |
| 闭包 / Block          | 原生                   | GCD Block    | ⚠️    | 后加，非一等   |
| image / live system | 强支持                  | ❌            | 完全阉割  |          |
| IDE 反射              | 系统级                  | ❌            | 工具链外置 |          |
| 强动态类型               | 默认                   | 可选（id）       | ⚠️    | 与静态妥协    |
| 消息多播                | 原生支持                 | ❌            | 留给框架  |          |

## 最本质的差异

> **Smalltalk 相信“系统是活的”；ObjC 相信“系统是要交付的”。**

| Smalltalk   | ObjC       |
| ----------- | ---------- |
| 程序 = 运行中的世界 | 程序 = 编译产物  |
| 方法不存在是对话    | 方法不存在是 bug |
| IDE 是语言的一部分 | IDE 是外部工具  |
| 一致性 > 性能    | 性能 > 一致性   |
